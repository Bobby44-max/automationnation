What Was Inherited vs. What Was Delivered

The original WordPress SQL dump existed, but it was far from a working product. Here's the reality of what was handed over versus what was built.

  • No workflow automation

What Was Delivered

  • Fully functional live website at eduprephub.com
  • Stripe payment processing with subscriptions
  • Clerk authentication with secure user management
  • Supabase database (PostgreSQL) — clean schema
  • n8n workflow automation on dedicated VPS

  • Email notification system
  • Complete purchase-to-access pipeline

🔎 Why "Having the SQL Dump" ≠ Having a Product

A SQL dump is raw data in a file — it is not a website, not a server, not a product. Think of it like having a box of parts from a totaled car. You technically "have the car," but you can't drive it anywhere.

To turn that dump into what EduPrep Hub is today required:

  • Standing up entirely new hosting infrastructure
  • Designing and implementing a new database schema (Supabase/PostgreSQL)
  • Building a payment system from zero (Stripe)
  • Building an authentication system from zero (Clerk)
  • Creating automated workflows for user provisioning (n8n on VPS)
  • Designing and building every customer-facing page
  • Setting up SEO, sitemaps, and search engine indexing
  • Testing and debugging every integration end-to-end

This is not a restoration. This is a ground-up rebuild on a modern stack.

Timeline

7-Week Build Timeline

A week-by-week breakdown of what was accomplished, by one person, while simultaneously learning the platforms.

Week 1
Assessment & Infrastructure
Evaluated the WordPress SQL dump — determined it was unusable in its current state. Provisioned Hostinger VPS, registered domain, set up DNS, and installed n8n. Made the call to rebuild on a modern stack.
Week 2
Database & Authentication
Created Supabase project. Designed database schema from scratch (users, orders, licenses, products). Set up Clerk authentication — sign-up, sign-in, session management, protected routes.
Week 3
Payment System
Created Stripe account with business verification. Configured products, pricing plans, and checkout sessions. Built webhook endpoint to capture payment events and trigger downstream automation.
Week 4
Automation Workflows
Built n8n workflows on VPS: Stripe webhook listener → user ID generation → MentorMind license provisioning → welcome email with 6-digit password. Full purchase-to-access pipeline operational.
Week 5
Website Build
Designed and built all customer-facing pages. Product listings, purchase flows, user dashboard, login integration. Made responsive for mobile and desktop.
Week 6
Integration Testing & SEO
End-to-end testing of the entire pipeline: visit → purchase → auto-provision → email → login → access. Submitted sitemap to Google Search Console (17 pages indexed). Fixed bugs and edge cases.
Week 7
Polish, Monitoring & Handoff
UI polish, error handling improvements, VPS monitoring setup. Final production verification. System fully operational and live at eduprephub.com.
